# docs/solutions/

Structured knowledge base of solved problems and established patterns. Each document captures a problem, root cause, solution, and prevention strategy.

Created with [Compound Engineering](https://github.com/compound-engineering) — each documented solution compounds the team's knowledge.

## Categories

- **integration-issues/** — Cross-layer synchronization, serialization, type evolution
- **logic-errors/** — Simulation bugs, balance issues, determinism problems
- **patterns/** — Cross-cutting patterns derived from multiple learnings

## Usage

Search by tags, module, or component in YAML frontmatter. Use `ce:compound` after solving a non-trivial problem to add new learnings. Use `ce:compound-refresh` periodically to keep docs in sync with the codebase.
